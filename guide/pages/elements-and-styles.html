Elements and Styles

We have provided styles for many possible componants and requirements. But if we have missed something simply add a feature request to the Github page.

Typography

Typographic styles

Various typographic styles are present to allow for different data elements and content types.

Colour is also used with data element typography to highlight monetary values, and currencies are displayed using css ::before elements.

Tables

Responsive tables

Tables are placed full width within the .full-panel container. They are also responsive, so are swipe-able on smaller screens.

Forms

Form elements all include validation alerts and focus states. They also have placement for icons to help suggest what type of content should be entered in each field. If the icon is an action such as search functionality, the icon is coloured blue and on the right. But if they icon is just illustrative, it is grey and on the left.

It is important to be careful not to over-use labels. Rather than having for example, a label for First name, and last name... simply include one label for 'full name', and use placeholders to show first name and last name. This helps make forms simpler to follow for the user.

Overlay windows

There are 2 different types of overlay windows. Each for a different purpose.

  • Modal windows
  • full screen

Modal window

Modal windows are used for smaller alerts or dialogues.

Full screen window

Full screen windows however are used for longer content which can be scroll-able

Data visualization

There are many different charts and infographics to choose from for displaying data. These are using the chartist.js plugin and more information on its use can be found on their website.

Iconography
